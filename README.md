# 📚 Libris — Платформа книжных рецензий

Данный репозиторий содержит код проекта Libris (Backend на FastAPI + Frontend на React). Вся работа и запуск проекта осуществляются строго через Docker.

## Первый запуск проекта

Чтобы развернуть проект с нуля, выполните следующие шаги по порядку:

### Шаг 1. Создание файлов окружения (.env)
В репозитории хранятся только шаблоны `.env.example`. Для работы нужны реальные `.env` файлы. 
Создайте их с помощью готового скрипта:
./setup_env.sh
Скрипт автоматически скопирует шаблоны и создаст backend/.env и frontend/.env.

### Шаг 2. Генерация SECRET_KEY
Для безопасности приложения необходимо задать уникальный секретный ключ для JWT-токенов.
Сгенерируйте ключ с помощью команды:
openssl rand -hex 32
Скопируйте полученную строку, откройте файл backend/.env и вставьте её в параметр SECRET_KEY=.

### Шаг 3. Запуск контейнеров
Запустите сборку и старт всех контейнеров в фоновом режиме:
docker compose up -d --build
Docker скачает необходимые образы, установит зависимости, соберет React-приложение и поднимет базу данных PostgreSQL и Redis.

### Шаг 4. Применение миграций (Alembic)
При старте контейнера бэкенда миграции должны накатываться автоматически. Однако, если вам нужно применить миграции вручную (например, при обновлении кода), выполните команду внутри работающего контейнера бэкенда:
docker compose exec backend alembic upgrade head

### Шаг 5. Доступ к сервисам
После успешного запуска проект будет доступен по следующим адресам:
- Frontend: http://localhost
- Backend API (Swagger): http://localhost:8000/docs

## Управление миграциями

Все команды для работы с базой данных должны выполняться внутри контейнера backend.

1. Создать новую миграцию (если вы изменили модели в коде):
docker compose exec backend alembic revision --autogenerate -m "название_миграции"

2. Накатить миграции (обновить базу до последней версии):
docker compose exec backend alembic upgrade head

3. Откатить последнюю миграцию (на один шаг назад):
docker compose exec backend alembic downgrade -1

## Полезные команды Docker

Пересобрать и перезапустить проект:
docker compose up -d --build

Остановить работу проекта (контейнеры остановятся, но данные в базе сохранятся):
docker compose stop

Остановить проект и полностью удалить контейнеры:
docker compose down

Остановить проект и удалить данные базы:
docker compose down -v

Посмотреть логи в реальном времени:
docker compose logs -f

Посмотреть логи конкретного сервиса:
docker compose logs -f backend
docker compose logs -f frontend

Перезапустить только бэкенд:
docker compose restart backend

Зайти внутрь контейнера бэкенда:
docker compose exec backend bash

Проверить статус контейнеров:
docker compose ps
